# Release Notes: Issue #1aeddeb (통합 count 조회 기능)

## 1. 개요
본 릴리스는 #1aeddeb "통합 count 조회 기능"에 대한 최종 구현 및 검증 결과를 포함합니다. 
기존의 개별 데이터 수집 기능을 넘어, 수집된 데이터를 통합하여 조회하고 대용량 데이터 환경에서도 안정적으로 동작할 수 있도록 페이지네이션 및 필터링 기능이 추가되었습니다.

- **Issue ID**: #1aeddeb
- **Release Version**: v1.1.0-rc1
- **Release Date**: 2026-02-12
- **Baseline Commit**: 5cb0a4eda1dee1ce50fd7a2730e3a9d856b0513e
- **Branch**: `feature/1aeddeb-integrated-count-view`

## 2. 주요 변경 사항
### 신규 기능 (New Features)
- **통합 Count 조회 API (`GET /api/v1/counts`)**: 여러 소스에서 수집된 count 정보를 한 번에 조회할 수 있는 인터페이스를 제공합니다.
- **페이지네이션 및 필터링**: 대량의 데이터(10만 건 이상)를 효율적으로 처리하기 위해 `limit`, `offset` 기반의 페이지네이션과 `external_id` 필터링 기능을 도입하였습니다.
- **권한 기반 접근 제어**: 조회 기능 사용을 위해 `query` 권한 검증 로직을 추가하여 보안을 강화하였습니다.

### 성능 최적화 (Optimization)
- **파일 스토리지 조회 최적화**: `bufio.Scanner`를 사용한 라인 단위 처리를 통해 메모리 사용량을 최소화하고, 필요한 데이터(limit) 확보 시 조기 종료되도록 개선하였습니다.
- **응답 속도**: 통합 테스트 결과, 평균 응답 시간 **8.97ms**(기준 100ms 미만)를 기록하여 성능 요구사항(QR-04)을 여유 있게 충족합니다.

## 3. 결함 수정 및 조치 (Bug Fixes)
- **권한 관리 수정**: 테스트 스크립트(`gen-token.go`)에서 조회 권한(`query`)이 누락되었던 문제를 해결하였습니다.
- **배포 환경 정리**: Kubernetes 배포 시 구버전 리소스(Deployment)가 신규 리소스(StatefulSet)와 트래픽 간섭을 일으키는 현상을 확인하고 수동 제거 및 조치하였습니다.
