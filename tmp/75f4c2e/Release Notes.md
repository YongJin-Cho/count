## 3. 주요 변경 사항 (Change Log)
- **웹 UI 추가**: Go 템플릿과 HTMX를 활용한 카운트 관리 UI 구현.
- **인증 강화**: 모든 UI 엔드포인트에 Bearer Token 기반 인증 적용 및 `auth-module` 통합.
- **에러 핸들링 개선**: HTMX의 `HX-Retarget` 헤더를 활용하여 사용자 친화적인 에러 메시지 표시 기능 구현.
- **통합 테스트 자동화**: API 및 UI 접근성을 검증하는 통합 테스트 스크립트(`integration-test.sh`) 구축.
- **표준화**: 중복 생성(409) 및 형식 오류(400) 등 HTTP 응답 규격 정규화.

## 4. 운영 영향 및 주의 사항
- **보안**: UI 접근 시 유효한 인증 토큰이 필요하며, `Session Storage`를 통해 관리됩니다.
- **성능**: SSR 구조로 인해 클라이언트 부담은 적으나, 서버 측 템플릿 렌더링 오버헤드를 모니터링해야 합니다. (현재 P95 10ms 미만으로 매우 양호)
- **배포**: 신규 이미지 반영을 위해 배포 시 `kubectl rollout restart`를 권장합니다.

## 5. 향후 개선 과제 (Backlog)
- 카운트 수정 취소 시 목록으로 자동 복귀하는 UX 개선.
- CI/CD 파이프라인에서 `latest` 태그 대신 Git 커밋 SHA를 활용한 이미지 태그 관리.
- Playwright 등을 활용한 브라우저 기반 UI 자동화 테스트 도입.
